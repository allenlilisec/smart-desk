# SmartDesk 智能服务平台 — 用户故事与验收标准

> 版本：v1.0（已冻结）　|　冻结日期：2026-06-14　|　编制：温知（产品经理）/ 沈思（业务分析）
> 配套：[《SmartDesk 产品需求说明书 PRD》](SmartDesk产品需求说明书PRD.md)
>
> **状态：已冻结（v1.0），随 PRD 一并冻结。** 验收标准用 Given/When/Then 表述，由测试团队（严谨 / 林桥）作为功能用例基线；含 ⚠ 的条目对应的 PRD §10 开放问题已在 PRD §10.1 裁决，相关 AC 以裁决结果为准更新（如 US-2.2 终态/超时不自动关闭、US-3.1 阈值 0.85 且一期建议模式、US-7.1/US-2.x 重开窗口 7 天等）。
>
> 格式约定：每个 Epic 对应 PRD 的能力域（F1~F7）。故事 `US-x.y` = 作为〈角色〉，我想要〈能力〉，以便〈价值〉。验收标准为可测的 AC（Acceptance Criteria）。

## Epic F1 · 认证与权限

### US-1.1 登录与会话
作为**任意用户**，我想要用账号登录，以便安全访问属于我的工单与功能。
- AC1 Given 有效凭证，When 登录，Then 返回 JWT 并进入对应角色首页。
- AC2 Given 无效凭证，When 登录，Then 返回 401 且不下发令牌，错误提示不泄露账号是否存在。
- AC3 Given 已登录，When 令牌过期，Then 受保护接口返回 401，前端引导重新登录 / 刷新。
- AC4 Given 登出，Then 会话失效，后续请求 401。
- ⚠ OQ-2：身份来源（自建 vs SSO/OIDC）影响登录实现。

### US-1.2 RBAC 鉴权
作为**系统**，我想要按角色校验每个受保护操作，以便实现最小权限。
- AC1 Given `requester`，When 访问他人工单或管理后台，Then 返回 403。
- AC2 Given `agent`，When 受理 / 处理被授权队列工单，Then 允许；越权操作 403。
- AC3 Given `manager`，When 访问看板只读数据，Then 允许；尝试修改工单，Then 403。
- AC4 Given `admin`，When 配置分类 / SLA / 用户角色，Then 允许。
- AC5 所有越权访问均记录审计日志。

### US-1.3 网关入口与限流
作为**系统**，我想要网关统一入口、聚合路由与限流，以便保护后端并统一鉴权。
- AC1 所有业务请求经网关鉴权后转发，未经鉴权直达后端被拒。
- AC2 Given 超过速率阈值，When 持续请求，Then 返回 429。
- AC3 网关对关键请求埋点审计（who/when/what/route）。

## Epic F2 · 工单生命周期

### US-2.1 结构化提单
作为**报单人**，我想要一键填写结构化工单并提交，以便快速、规范地发起请求。
- AC1 Given 必填项（标题、描述）齐全，When 提交，Then 创建成功并返回工单号，状态=`新建`。
- AC2 Given 缺必填项，When 提交，Then 行内校验拦截并提示。
- AC3 提交后可附加附件（见 US-2.7）。
- AC4 建单成功即触发自动分类 / 定级 / 相似推荐（异步），AI 不可用时**建单仍成功**（降级）。

### US-2.2 工单状态流转
作为**坐席**，我想要按规范状态机推进工单，以便处理过程可控可追溯。
- AC1 状态流转遵循：`新建→受理→处理中→待用户→已解决→已关闭`，非法跃迁被拒。
- AC2 置`待用户`时 SLA 计时暂停；报单人回复后自动恢复并回到`处理中`。
- AC3 每次状态变更写入时间线（操作人、时间、前后状态）。
- AC4 状态变更操作幂等，重复提交不产生重复记录。
- ⚠ OQ-8：是否含`挂起`/`取消`、`待用户`超时自动关闭，以最终裁决为准。

### US-2.3 分派与改派
作为**组长 / 系统**，我想要把工单分派给合适坐席，以便及时处理。
- AC1 组长可手工分派 / 改派给本组坐席。
- AC2 支持按分类 / 技能组自动分派（规则可配置）。
- AC3 改派记录于时间线并通知新责任人。

### US-2.4 评论与内部备注
作为**坐席**，我想要区分对外评论与内部备注，以便协作同时不泄露内部信息。
- AC1 对外评论报单人可见并触发通知。
- AC2 内部备注仅 `agent`/`lead`/`manager` 可见，`requester` 不可见、接口不返回。
- AC3 评论支持 @提及（见 US-5.2）。

### US-2.5 SLA 计时与超时
作为**系统**，我想要按优先级计 SLA 并在临近 / 超时预警，以便守住时限。
- AC1 建单 / 受理按优先级启动响应与解决计时。
- AC2 `待用户`期间计时暂停，恢复后续计。
- AC3 临近阈值发布预警事件并通知坐席 + 组长；超时标记`已超时`。
- ⚠ OQ-1：阈值表由业务提供。

### US-2.6 工单关联与合并
作为**坐席**，我想要把重复 / 相关工单关联或合并，以便避免重复处理。
- AC1 可关联两张工单并在时间线互显。
- AC2 合并后子单指向主单，状态同步，报单人收到合并通知。

### US-2.7 附件
作为**报单人 / 坐席**，我想要上传 / 下载附件，以便补充上下文。
- AC1 上传受大小 / 类型白名单限制，超限被拒并提示。
- AC2 附件下载需鉴权，越权 403。
- ⚠ OQ-9：大小 / 类型 / 存储方式待定。

### US-2.8 时间线与审计
作为**坐席 / 管理者**，我想要查看工单完整时间线，以便追溯处理过程。
- AC1 时间线按时间正序展示所有关键事件（建单、分派、状态、评论、SLA、关闭）。
- AC2 审计记录不可被普通用户篡改 / 删除。

## Epic F3 · 智能分析

### US-3.1 自动分类
作为**报单人 / 坐席**，我想要系统自动建议工单分类，以便减少人工打标。
- AC1 建单后系统给出分类预测与置信度。
- AC2 置信度 ≥ 阈值时自动填充，< 阈值时仅作建议，坐席可一键采纳或修改。
- AC3 人工纠偏结果被记录（供模型迭代）。
- ⚠ OQ-4：分类体系、冷启动数据、阈值待裁决。

### US-3.2 优先级建议
作为**坐席**，我想要系统建议优先级，以便快速定级。
- AC1 系统给出优先级建议，坐席可调整。
- AC2 调整后 SLA 计时按新优先级重算。

### US-3.3 相似工单推荐
作为**坐席**，我想要看到相似历史工单与解决方案，以便复用经验缩短处理时间。
- AC1 打开工单时展示 Top-N 相似工单（带相似度 / 摘要）。
- AC2 点击可跳转查看历史解决方案。
- AC3 推荐计算不阻塞工单详情主加载（异步 / 懒加载）。
- ⚠ OQ-5：检索路线（关键词 / 向量）与选型待裁决。

### US-3.4 统计聚合
作为**系统**，我想要聚合工单指标，以便支撑看板。
- AC1 提供按时间 / 分类 / 坐席 / 优先级的出量、SLA 达成、时长等聚合接口。
- AC2 聚合数据与明细一致（同口径校验）。

## Epic F4 · 通知与事件

### US-4.1 领域事件
作为**系统**，我想要把关键业务动作发布为领域事件，以便解耦通知与分析。
- AC1 建单 / 分派 / 状态变更 / 评论 / SLA 预警 / 解决 / 关闭等均发布事件。
- AC2 事件含统一 schema（事件类型、工单 ID、时间、操作人、载荷）。
- AC3 事件至少一次送达，消费方幂等处理。

### US-4.2 站内通知
作为**用户**，我想要在站内通知中心看到与我相关的更新，以便及时跟进。
- AC1 相关事件生成站内通知，区分未读 / 已读。
- AC2 点击通知跳转到对应工单。

### US-4.3 邮件通知
作为**用户**，我想要关键事件收到邮件，以便离线也能知晓。
- AC1 关键事件（分派给我、解决、超时）发送模板化邮件。
- AC2 邮件发送失败可重试并记录。

### US-4.4 通知策略
作为**管理员**，我想要配置通知策略，以便控制噪声。
- AC1 admin 可按角色 / 事件类型开关订阅。
- AC2 策略变更即时生效。

## Epic F5 · 协作

### US-5.1 转派与升级
作为**坐席**，我想要转派或升级工单，以便交给更合适的人。
- AC1 转派需选择目标坐席 / 组并记录原因。
- AC2 升级上报组长 / 管理者并通知。

### US-5.2 @提及
作为**坐席**，我想要 @同事，以便拉入协作。
- AC1 评论 / 备注中 @某人触发其通知。
- AC2 被 @者无权限查看该工单时不发送（鉴权校验）。

### US-5.3 关注 / 跟随
作为**用户**，我想要关注一张工单，以便接收其更新。
- AC1 关注后该工单更新进入我的通知。
- AC2 可取消关注。

## Epic F6 · 看板与报表

### US-6.1 坐席工作台
作为**坐席**，我想要看到我的待办与即将超时工单，以便优先处理。
- AC1 展示「我的待办」「即将超时」「按状态分组」。
- AC2 即将超时项高亮并可排序。

### US-6.2 管理看板
作为**管理者 / 组长**，我想要实时看板，以便掌握 SLA 与积压。
- AC1 展示 SLA 达成率、积压趋势、坐席工作量、分类分布。
- AC2 支持时间范围 / 分类 / 坐席筛选。
- AC3 数据近实时（延迟在约定范围内）。

### US-6.3 质量视图
作为**管理者**，我想要看重开率 / 超时单 /（可选）满意度，以便评估质量。
- AC1 展示重开率、超时单列表。
- AC2 ⚠ OQ-12：CSAT 是否纳入。

### US-6.4 导出
作为**管理者**，我想要导出报表，以便线下分析。
- AC1 可按当前筛选导出 CSV。

## Epic F7 · 三端前端

### US-7.1 报单人门户
作为**报单人**，我想要简洁的提单与「我的工单」视图，以便低门槛使用。
- AC1 首页可一键提单；「我的工单」按状态查看与追加评论 / 确认解决 / 重开。
- ⚠ OQ-13：重开窗口期。

### US-7.2 坐席工作台
作为**坐席**，我想要队列 + 处理一体化界面，以便高效处理。
- AC1 左队列右详情，详情含时间线、评论 / 备注、相似推荐、状态操作。

### US-7.3 管理后台
作为**管理员**，我想要配置分类体系、SLA 策略、用户 / 角色、通知策略，以便运营平台。
- AC1 分类体系增删改查（树形）。
- AC2 SLA 策略按优先级配置时限。
- AC3 用户 / 角色管理（RBAC）。
- AC4 通知策略配置。
