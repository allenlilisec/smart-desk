# SmartDesk 集体检视门禁清单

> 版本：v1.0　|　生效日期：2026-06-18　|　维护：质量管理团队（韩衡）　|　裁决：CTO
>
> 本清单为 SmartDesk 项目 PR 进入合入前的集体检视门禁，检视人须逐条核对并记录结果。
> 清单变更须经 CTO 裁决后方可生效。

---

## 通用说明

- **触发时机**：PR 状态转为 `in_review` 时，责任人须在 PR 描述中声明本清单相关项的核查结果；检视人逐条核验。
- **不通过处理**：任一必检项未通过，检视人应将 PR 驳回（request changes），责任人修正后重新发起检视。
- **记录方式**：建议在各 PR 描述末尾复制对应检查项的声明模板并勾选。

---

## 必检项

### G-X：跨服务调用路径与 OpenAPI 契约一致

- **生效日期**：2026-06-18
- **来源事件**：RB-2026-0617-01（见 [SUP-313](mention://issue/5944bf76-fa2c-42e2-9442-15016690e731)：SUP-303 端点路径漂移，smartdesk-insight PR #14 使用 `/config/users/{userId}` 与最终契约 `/internal/users/{userId}` 不一致）
- **触发范围**：任何**改动或新增跨服务调用**的 PR 进入 `in_review` 时强制核对。仅服务内部改动可豁免，但须评审人显式确认无跨服务调用变更。

**自动化防线（已落地）**：

- CI 工作流 `.github/workflows/api-contract-check.yml` 在 PR / push 到 `main` 时自动运行；
- `scripts/api_contract_check.py` 扫描调用方源码中的 HTTP 请求路径，与 `src/openapi/*.yaml` 做模板匹配；
- 自动拦截：路径未定义、HTTP 方法不一致、调用已废弃路径（如 `/config/users/{userId}` 已标记为 `smartdesk-insight` 的废弃路径）。

**通过标准（逐条核对，缺一不可）**：

1. CI 中 `api-contract-check` 检查通过；
2. 调用方代码中的外部端点路径，与 `src/openapi/*.yaml` 中提供方契约**逐字符一致**（方法 + 路径 + 路径参数名）；
3. 调用方所依赖的契约版本，是否为**当前生效版本**（非已废弃 PR 的旧路径）——评审人须确认 submodule/契约指针指向最新裁决落地的 commit；
4. 若本 PR 变更了契约，则**契约文件、提供方实现、调用方实现、api-contract-check 用例**四者必须同 PR 或同批次同步更新，不允许只改其中一处；
5. 对于自动检查未覆盖场景（动态拼接路径、非标准 HTTP 客户端、新增消费者未纳入配置等），检视人须手工补充核对。

**未通过动作**：驳回，不得合入。

**声明模板**（建议附在 PR 描述末尾）：

```markdown
### 跨服务调用路径一致性声明

- [ ] 本 PR 改动/新增了跨服务调用
- [ ] CI 中 `api-contract-check` 检查通过
- [ ] 调用方代码中的外部端点路径与 `src/openapi/*.yaml` 最终契约逐字符一致
- [ ] 所依赖的契约版本为当前生效版本（submodule/契约指针已指向最新裁决落地的 commit）
- [ ] 若本 PR 变更了契约，契约文件、提供方实现、调用方实现、api-contract-check 用例已同批次同步更新
- [ ] 检视人已逐条核验并通过（含自动检查未覆盖场景）
```
