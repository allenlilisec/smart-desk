# SmartDesk 集体检视门禁清单

> 版本：v1.2　|　生效日期：2026-06-20　|　维护：质量管理团队　|　裁决：CTO
>
> 本清单为 SmartDesk 项目 PR 进入合入前的集体检视门禁，检视人须逐条核对并记录结果。
> 清单变更须经 CTO 裁决后方可生效。

---

## 通用说明

- **触发时机**：PR 状态转为 `in_review` 时，责任人须在 PR 描述中声明本清单相关项的核查结果；检视人逐条核验。
- **不通过处理**：任一必检项未通过，检视人应将 PR 驳回（request changes），责任人修正后重新发起检视。
- **记录方式**：建议在各 PR 描述末尾复制对应检查项的声明模板并勾选。

---

## 必检项

### G-X-E：跨服务契约一致性双检

- **生效日期**：2026-06-20
- **来源事件**：RB-2026-0620-01
- **触发范围**：任何**改动或新增跨服务调用或跨服务事件发射**的 PR 进入 `in_review` 时强制核对。仅服务内部改动可豁免，但须评审人显式确认无跨服务调用/事件变更。
- **说明**：G-X-E 在原有 G-X 基础上扩展为「双检」——既检 HTTP 端点路径一致性（E.1），也检事件 payload schema 一致性（E.2）。原 G-X 内容归入 E.1。

#### E.1 HTTP 端点路径一致性（原 G-X）

**自动化防线（已落地）**：

- CI 工作流 `.github/workflows/api-contract-check.yml` 在 PR / push 到 `main` 时自动运行；
- `scripts/api_contract_check.py` 扫描调用方源码中的 HTTP 请求路径，与 `src/openapi/*.yaml` 做模板匹配；
- 自动拦截：路径未定义、HTTP 方法不一致、调用已废弃路径（如 `/config/users/{userId}` 已标记为 `smartdesk-insight` 的废弃路径）。

**通过标准（逐条核对，缺一不可）**：

1. CI 中 `api-contract-check` 检查通过；
2. 调用方代码中的外部端点路径，与 `src/openapi/*.yaml` 中提供方契约**逐字符一致**（方法 + 路径 + 路径参数名）；
3. 调用方所依赖的契约版本，是否为**当前生效版本**（非已废弃 PR 的旧路径）——评审人须确认 submodule/契约指针指向最新裁决落地的 commit；
4. 若本 PR 变更了契约，则**契约文件、提供方实现、调用方实现、api-contract-check 用例**四者必须同 PR 或同批次同步更新，不允许只改其中一处；
5. 对于自动检查未覆盖场景（动态拼接路径、非标准 HTTP 客户端、新增消费者未纳入配置等），检视人须手工补充核对。

#### E.2 事件 payload schema 一致性（新增）

**自动化防线（已落地）**：

- CI 工作流运行新增脚本 `scripts/event_schema_check.py`，校验事件发布方源码中的事件 payload 与事件契约定义文件（如 `src/events/*.yaml`，具体以 core 契约唯一事实源为准）的 schema 一致；
- core 契约回归测试覆盖事件 payload 的关键字段、类型及必填约束；
- 自动拦截：schema 未定义、字段类型/必填约束不一致、调用已废弃/未生效的 schema 版本。

**通过标准（逐条核对，缺一不可）**：

1. CI 中 `scripts/event_schema_check.py` 检查通过；
2. 事件发布方代码中的 payload 字段、类型、必填约束，与事件契约定义文件**逐字符一致**；
3. 所依赖的事件 schema 版本为当前生效版本——评审人须确认契约指针/版本号指向最新裁决落地的 commit；
4. 若本 PR 变更了事件 schema，则**schema 文件、发布方实现、消费方实现、`event_schema_check.py` 用例、core 契约回归测试**五者必须同 PR 或同批次同步更新，不允许只改其中一处；
5. 对于自动检查未覆盖场景（动态构造 payload、非标准事件客户端、新增事件类型未纳入配置等），检视人须手工补充核对。

**未通过动作**：E.1、E.2 任一子检查未通过，检视人应驳回（request changes），不得合入。

**声明模板**（建议附在 PR 描述末尾）：

```markdown
### 跨服务契约一致性双检声明（G-X-E）

- [ ] 本 PR 改动/新增了跨服务调用或跨服务事件发射
- [ ] E.1 CI 中 `api-contract-check` 检查通过
- [ ] E.1 调用方代码中的外部端点路径与 `src/openapi/*.yaml` 最终契约逐字符一致
- [ ] E.1 所依赖的 OpenAPI 契约版本为当前生效版本（submodule/契约指针已指向最新裁决落地的 commit）
- [ ] E.1 若本 PR 变更了 OpenAPI 契约，契约文件、提供方实现、调用方实现、api-contract-check 用例已同批次同步更新
- [ ] E.2 CI 中 `scripts/event_schema_check.py` 检查通过
- [ ] E.2 事件发布方代码中的 payload 与事件契约定义文件最终 schema 逐字符一致
- [ ] E.2 所依赖的事件 schema 版本为当前生效版本
- [ ] E.2 若本 PR 变更了事件 schema，schema 文件、发布方实现、消费方实现、event_schema_check 用例、core 契约回归测试已同批次同步更新
- [ ] 检视人已逐条核验并通过（含自动检查未覆盖场景）
```
