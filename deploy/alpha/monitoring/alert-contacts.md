# Alpha/Beta 告警联系人链与升级路径

> 对齐 [`specs/发布监控告警基线.md`](../../specs/发布监控告警基线.md) §6 与 [`specs/回滚预案.md`](../../specs/回滚预案.md) §6。  
> **环境策略**：安全测试环境全量部署，**无金丝雀放量**；「异常即暂停」适用于部署窗口与运行态观测。

## 1. 角色与职责

| 角色 | 负责人 | Alpha/Beta 职责 |
|---|---|---|
| 发布经理 | 白帆 | 部署 Go/No-Go、暂停/回滚裁决、LKG 清单签字 |
| 灰度/回滚/监控 | 万全 | 探针维护、观测快照、回滚执行、dry-run 记录 |
| 人类决策 | Allen Li（CTO） | 重大回滚通报；安全测试授权链路（`multica-test@hotmail.com`） |
| 后端 on-call | 石磊 | core/gateway 实现侧故障 |
| 智能 on-call | 苏睿 | insight/事件消费故障 |
| 安全 | 武安 | 渗透测试执行；鉴权/越权异常确认 |
| PMO | 程远 | 安全验收主线跟踪（SUP-184 回流） |

## 2. 通报链路

```
观测异常（探活失败 / H-* / R-* / P0）
    │
    ├─ 暂停（H-* / P1）──► 万全（立即）──► 白帆（≤10 min）
    │                      └── 冻结部署/变更，Issue 评论通报
    │
    ├─ 强制回滚（R-* / P0）──► 万全执行 §4 回滚 ──► 白帆确认恢复
    │                           └── CTO 邮件（重大事故 / 数据风险）
    │
    └─ 安全事件（R-5）──► 武安确认 ──► 白帆裁决回滚 ──► 程远回流 SUP-184
```

## 3. Alpha/Beta 渠道

| 级别 | 渠道 | 响应时限 |
|---|---|---|
| P0 / R-* | Issue 评论 + 邮件 `multica-test@hotmail.com` | 5 min 内决策，15 min 内完成回滚（RTO） |
| P1 / H-* | Issue 评论（SUP-203 / 本子任务） | 10 min 内研判 |
| P2 | 探活日志 / dry-run 记录归档 | 下一部署窗口前复核 |

## 4. 异常即暂停（本环境）

| 条件 | 动作 | 通知 |
|---|---|---|
| 任一服务 `/readyz` 连续 2 次非 200（H-3） | **暂停**后续部署步骤 | 万全 → 白帆 |
| gateway 5xx > 1% 持续 2 min（H-1） | **暂停** | 万全 → 白帆 |
| 建单成功率 < 99%（R-2） | **强制回滚**至 LKG | 万全 → 白帆 → CTO |
| 鉴权失败率 > 5% 持续 2 min（R-3） | **强制回滚** | 万全 → 白帆 → 武安（若安全相关） |

> 阈值与 [`specs/回滚预案.md`](../../specs/回滚预案.md) §2、`specs/发布监控告警基线.md` §5 保持一致。

## 5. 升级路径

1. **L1** 万全：探活、快照、执行回滚脚本  
2. **L2** 白帆：暂停/回滚裁决、协调模块 on-call  
3. **L3** 模块 Leader：石磊（core/gateway）、苏睿（insight）、江颜（web）  
4. **L4** CTO：重大事故、超 RTO、数据丢失风险  
5. **L5** PMO 程远：阻塞安全验收主线时当日升级
