## 1. 测试目标

验证四类告警场景可被正确触发并生成标准 payload：

1. workflow 失败告警（`deploy-canary.yml` / `rollback.yml` 步骤失败）。
2. 人工确认超时告警（GitHub Environment required reviewer 超时）。
3. 阶段推进提醒（C1/C2/C3 与 rollback 成功）。
4. 镜像/部署异常告警（构建失败、GHCR 推送失败、compose 脚本非零退出）。

## 3. GitHub Actions 触发测试

### 3.1 测试工作流

路径：`.github/workflows/alert-test.yml`

触发方式：仓库 Actions → `Alert Trigger Tests` → `Run workflow`。

### 3.2 测试用例与结果

| 用例 | 场景 | 预期告警 | GitHub 运行结果 | 备注 |
|---|---|---|---|---|
| T1 | 成功/推进通知 | `success` 级别 | 待环境配置后触发 | 校验 payload 字段与 artifact 上传 |
| T2 | workflow 步骤失败 | `P0` 失败告警 | 待环境配置后触发 | `simulate-failure` job 失败后 `notify-failure` 应运行 |
| T3 | 镜像/部署异常 | `P0` 失败告警 | 待环境配置后触发 | 模拟 GHCR 推送失败 |
| T4 | 人工确认超时 | `P1` 超时告警 | 待环境配置后触发 | 需先配置 `canary-gate-c1` required reviewers，然后不审批触发 `deploy-canary.yml` |
| T5 | C1 部署成功 | `success` 推进提醒 | 待环境配置后触发 | 配置 Webhook 后触发 `deploy-canary.yml` stage=c1 |
| T6 | rollback 成功 | `success` 回滚完成 | 待环境配置后触发 | 触发 `rollback.yml` lkg_tag=latest |

> 当前阶段：本地静态验证已通过；GitHub 端实际触发需在仓库配置 `NOTIFY_WEBHOOK_URL` Secret 与 `canary-gate-*` Environment 后补录。

## 4. 已知限制与后续动作

1. **Webhook 未配置时仅记录日志**：这是设计行为，便于在未接入企业 IM 前完成调试。
